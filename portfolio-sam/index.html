Recent Projects

Project 1

Weather-Forecasting

This Flask web application provides a simple weather forecast service. Users can input a city name, and upon submitting the form, the app fetches real-time weather data from the OpenWeatherMap API. The front end consists of two HTML pages: the index page with a form for city input, and the weather page displaying the retrieved weather information. The Flask backend handles requests, retrieves data from the API, and sends it to the frontend for dynamic display. The project showcases the integration of Flask, HTML templates, and external API calls to create a basic but functional weather forecast application.

Project 2

Personal-Finance-Tracker

This JavaScript code manages a basic transaction tracking system for a web application. It includes functionalities to display transactions, update the balance, and add new transactions. The transactions are stored in an array, and their details are presented in a list on the webpage. The balance is dynamically calculated based on the transaction amounts. Users can input new transactions through a form, and the data is stored in the array. Additionally, the transactions are saved to local storage, allowing data persistence between page reloads. The code employs event listeners to handle form submissions and page unloading.

Project 3

ExcelOpsManager

This Node.js code uses the exceljs library to interact with Excel files. It defines functions to save, create, read, update, and delete entries in an Excel sheet. The saveWorkbook function writes the workbook to a file ('data.xlsx'). The createEntry function adds a new row to the sheet with provided data. The readData function retrieves all rows from the sheet as an array of objects. The updateEntry function updates values in a specified row with new data. The deleteEntry function removes a specified row from the sheet. The example usage section demonstrates how to create, read, update, and delete entries in the Excel sheet.

Project 4

DynamicFormVista

This Node.js and Express application connects to a MongoDB database, handling form data. It defines a MongoDB schema for form entries, creates an Express server, and establishes routes for submitting and retrieving form data. The server listens on port 5000, and upon successful MongoDB connection, it starts serving requests. The '/api/submit' endpoint allows clients to submit form data, which is saved to MongoDB. The '/api/formData' endpoint retrieves all form data from the database. Error handling is implemented, providing appropriate responses in case of success or failure.

Project 5

EcommBotInteract.

This Python script integrates a Machine Learning chatbot (trained on an FAQ dataset) into a Tkinter GUI for a conversational interface. The chatbot processes user input, predicts responses, and dynamically updates the conversation with user and chatbot messages, each accompanied by corresponding images. Additionally, a Flask web server serves HTML templates for different sections of an e-commerce website, and an API endpoint allows interaction with the chatbot. The script is multithreaded to run both the Flask app and Tkinter GUI concurrently
